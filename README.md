Explique, com suas próprias palavras e de forma completa:
1.	O que é uma View no Android Framework?
2.	O que é um ViewGroup no contexto da interface Android?
3.	Qual é a função dos LayoutParams dentro de um ViewGroup?
4.	Por que diferentes ViewGroups, como LinearLayout, ConstraintLayout e FrameLayout, utilizam subclasses específicas de LayoutParams?
      
      Respostas:
      1-	Uma view é um componente visual da interface no aAndroid, é um elemento básico, que a gente pode visualizar e interagir na tela do app, como textos, botões etc.
      2-	No Android, um ViewGroup é um contêiner especial que funciona como um pai para outros componentes visuais, conhecidos como filhos, mas que no caso são as views. É meio que uma caixa de views, que pode ser de vários tipos, e serve para organizar.
      3-	Eles servem para definir paramêtros de molde e visualização dos componentes.
      4-	É porque cada um desses tipos tem um jeito diferente de organizar os compontentes, tendo que ter códigos de parâmetros diferentes, que sigam esse método.
